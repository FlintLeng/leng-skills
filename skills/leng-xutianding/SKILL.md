# 虚天鼎 (Leng Xutianding)

## 触发条件

当用户输入 **"虚天鼎"** 时，执行文件收集整理流程。

## 执行流程

### 1. 确定存储位置

默认存储路径：`~/Documents/虚天鼎/`，按日期创建子文件夹：
- 路径格式：`~/Documents/虚天鼎/YYYY-MM-DD/`
- 如果文件夹不存在，自动创建

### 2. 识别输入类型

根据用户提交的内容，识别以下类型：

| 类型 | 识别方式 | 处理方法 |
|------|----------|----------|
| 本地文件 | 文件路径或文件附件 | 复制到目标文件夹 |
| 网页URL | http/https 开头的链接 | 抓取网页内容，保存为 .md 或 .html |
| 飞书文档 | feishu.cn 或 larksuite.com 链接 | 使用飞书技能读取内容，保存为 .md |
| 飞书文件 | 飞书云盘文件链接 | 下载到目标文件夹 |
| 文本内容 | 纯文本段落 | 保存为 .txt 文件 |

### 3. 执行收集

对每个输入项执行收集操作：

**本地文件：**
```bash
cp <源文件路径> <目标文件夹>/
```

**网页URL：**
- 使用 `web_fetch` 工具抓取网页内容
- 保存为 markdown 格式：`<标题>.md`
- 文件名使用时间戳+序号避免冲突

**飞书文档：**
- 使用 `feishu-doc` 技能读取文档内容
- 保存为 markdown 格式

**飞书云盘文件：**
- 使用 `feishu-drive` 技能下载文件
- 保存到目标文件夹

**文本内容：**
- 直接保存为 `.txt` 文件
- 文件名格式：`文本_<时间戳>_<序号>.txt`

### 4. 错误处理

遇到以下情况时**跳过该项，继续处理下一个**：
- 文件不存在或无法访问
- 网页访问受限、超时、需要登录
- 飞书文档无权限访问
- 下载失败
- 任何其他执行错误

**重要：** 不因单个项目失败而中断整个流程，确保最终返回执行详情。

### 5. 返回执行详情

收集完成后，返回详细报告：

```
📊 虚天鼎 - 文件收集报告

【存储位置】
~/Documents/虚天鼎/2026-03-21/

【收集结果】
| 项目 | 类型 | 状态 | 文件名 |
|------|------|------|--------|
| ... | ... | ✅/⚠️/❌ | ... |

【统计】
- 成功：X 项
- 跳过：Y 项
- 失败：Z 项

【失败详情】（如有）
- 项目名：失败原因
```

## 使用示例

```
用户: 虚天鼎
      https://www.example.com/article
      /Users/xxx/Documents/report.pdf
      这是一段需要保存的文本内容
```

输出：收集所有项目到 `~/Documents/虚天鼎/2026-03-21/`，返回执行详情。

## 注意事项

1. **文件命名**：避免文件名冲突，使用时间戳和序号
2. **编码处理**：确保中文文件名和内容正确保存
3. **大文件**：大文件下载可能耗时较长，设置合理超时
4. **隐私安全**：不保存敏感信息（密码、密钥等）
